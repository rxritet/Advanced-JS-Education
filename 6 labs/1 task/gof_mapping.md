# GoF Class/Object Mapping for Lab 6.1

- SNIPPET_01 — Singleton — Object
  - В GoF‑таблице Singleton отмечен как Object‑уровень: важна глобальная точка доступа к экземпляру и его состояние, а не наследование или фабричный метод класса.

- SNIPPET_02 — Observer — Object
  - Observer в таблице относится к Object, так как общение строится через композицию подписчиков и издателя. Логика в том, как объекты связаны и оповещают друг друга.

- SNIPPET_03 — Facade — Object
  - Facade — объектный паттерн: он соединяет и прячет несколько готовых объектов подсистемы за одним фасадом. Мы работаем с комбинацией объектов, а не с иерархией классов.

- SNIPPET_04 — Factory Method — Class
  - Factory Method помечен как Class, потому что использует полиморфизм и, как правило, переопределяемые методы для создания подклассов, фокусируясь на уровне классовой иерархии.

- SNIPPET_05 — Decorator — Object
  - Decorator — объектный: поведение добавляется за счёт оборачивания одного объекта другим, используя композицию. Всё основано на связях объектов, а не на абстрактных суперклассах.

- SNIPPET_06 — Prototype — Object
  - Prototype относится к Object, так как опирается на клонирование конкретного экземпляра. Центр паттерна — уже созданный объект-прототип, который копируется, а не абстрактный класс.

- SNIPPET_07 — Chain of Responsibility — Object
  - Цепочка ответственности — объектный паттерн: запрос последовательно передаётся от объекта к объекту. Поведение возникает из построенной цепочки объектов, а не из схемы наследования.

- SNIPPET_08 — Command — Object
  - Command в таблице — Object: каждая команда — отдельный объект, который инкапсулирует действие. История и повторное выполнение основаны на коллекции этих объектов.

- SNIPPET_09 — Flyweight — Object
  - Flyweight — объектный структурный паттерн: он организует шаринг общего состояния между объектами через отдельные экземпляры flyweight и ссылки на них.

- SNIPPET_10 — Mediator — Object
  - Mediator — Object, поскольку центральный посредник — это отдельный объект, который управляет связями и взаимодействием между другими объектами-участниками.
