## 4. Что вы узнали

1. **SSR vs CSR разница по времени**: понял, что FCP достигается сразу при получении HTML от сервера, а TTI — только после выполнения JS; искусственный `setTimeout(1000)` в клиентском коде наглядно демонстрирует этот gap.
2. **XSS-защита через JSON.stringify**: `JSON.stringify` автоматически экранирует символы `<`, `>`, `&` — это предотвращает инъекцию скриптов через данные пользователя в SSR-шаблонах.
3. **Closure как инструмент изоляции**: переменная `count` внутри closure функции `counter` недоступна снаружи — это ключевой принцип Islands Architecture, предотвращающий глобальное загрязнение.
4. **IntersectionObserver для lazy hydration**: islands активируются только при попадании в viewport (`threshold: 0.1`), что снижает объём выполняемого JS при первой загрузке страницы.
